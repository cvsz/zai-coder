# v0.1.3 Safe Runner and Policy Registry Regression Tests

## Scope
Added regression test suites to guarantee robust validation of dangerous and safe command invocations under the standard `SafetyPolicy` class, fallback policies inside `PolicyLoader`, and file staging constraints defined by `repo_policy`.

## Modules Tested
* `zai_coder.core.safety.SafetyPolicy`
* `zai_coder.github_ready_core.repo_policy`
* `zai_coder.core.policy_loader.PolicyLoader`

## Dangerous Command Patterns Covered
* `git add .`
* `git add -A`
* `--no-verify`
* `git push --force` / `-f` / `--force-with-lease`
* `rm -rf /`
* `sudo rm -rf`
* Environment access via `cat .env`

## Safe Command Allowlist Cases Covered
* `git status --short`
* `git diff --stat`
* `python3 -m pytest -q`
* `make repo-check`

## Path/Staging Exclusions Covered
* Protected project directories (`apps/zlms/`)
* Standard build and dependencies paths (`dist/`, `node_modules/`, `.pytest_cache/`, `__pycache__/`)
* Dangerous file endings (`.zip`, `.tgz`, `.db`, `.sqlite3`, `.pyc`)
* Environment and config files (`.env`, `secrets.json`)

## Fixes Made
No modifications were required as the current code successfully blocks all target patterns.

## Compatibility Notes
* No existing behavior modified.
* Fail-closed behavior for unknown policies validated successfully.

## Validation Results
* Target test suite passed warning-free.
* All baseline test gates verified clean.
* No changes to `pyproject.toml` version metadata.
