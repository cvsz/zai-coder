# v0.1.2 Package and Release Gate Hardening

## Baseline Behavior
- Prior to hardening, `package-check.sh` only verified that the `.tar.gz` archive existed and did basic content inspection.
- It did not verify `.zip` archives.
- It did not verify checksum signature matching via `sha256sum -c`.
- It did not validate `RELEASE_MANIFEST.json` format or match its metadata to the authoritative `pyproject.toml` version.

## Package-Check Improvements
1. **Autoritative Version Parsing**: Uses Python's `tomllib` to parse the version from `pyproject.toml`.
2. **Artifact Existence Checks**: Fails if any expected artifact (`.tar.gz`, `.zip`, `.tar.gz.sha256`, `.zip.sha256`, `RELEASE_MANIFEST.json`) is missing.
3. **Checksum Verification**: Executes `sha256sum -c` against both the `.tar.gz.sha256` and `.zip.sha256` files.
4. **Manifest JSON Validation**: Checks that `RELEASE_MANIFEST.json` is well-formed JSON using Python's `json` library, and asserts that its version matches `pyproject.toml` and contains both archives.
5. **Output Structure**: Prints a compact JSON-like result mapping the verification status, version, and artifacts details:
   ```json
   {
     "ok": true,
     "version": "0.1.1",
     "artifacts": [
       {"name": "zai-coder-standalone-0.1.1.tar.gz", "size_bytes": 1472885},
       {"name": "zai-coder-standalone-0.1.1.zip", "size_bytes": 2242168}
     ]
   }
   ```
