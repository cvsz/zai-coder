# v0.1.2 Warning Cleanup

## Baseline Warnings (v0.1.1)
- **Count**: 2 warnings
- **Warning 1**: `PytestCollectionWarning` — `TestCase` dataclass in `zai_coder/quality_assurance_test_lab/models.py` was being collected as a test class because its name starts with `Test`.
- **Warning 2**: `PytestReturnNotNoneWarning` — `test_matrix` function exported from `core.py` via wildcard import into the test module was collected by pytest as a test function and returned a list.

## Fix Approach

### Fix 1: TestCase — PytestCollectionWarning
Added `__test__ = False` class attribute to `TestCase` in `models.py`. This is the standard, minimal pytest opt-out pattern for production dataclasses whose names start with `Test`.

### Fix 2: test_matrix → get_test_matrix — PytestReturnNotNoneWarning
Renamed `test_matrix` to `get_test_matrix` in `core.py`. Updated all internal callers and the routes module. Removed the wildcard `import *` from the test file and replaced it with an explicit import list. This eliminates the `test_matrix` symbol from the test module namespace entirely.
