# V0.1.3 Operator Release Gate Checklist

## 1. Pre-release Branch Gate
- [ ] Ensure `chore/v0.1.3-final-readiness` is merged to main
- [ ] Create `release/v0.1.3` branch from main
- [ ] Verify version remains `0.1.2` at branch start

## 2. Version Bump Gate
- [ ] Bump version to `0.1.3` in `pyproject.toml`
- [ ] Bump version in `zai_coder/__init__.py`
- [ ] Commit version bump (`chore(release): bump version to v0.1.3`)

## 3. Package Build Gate
- [ ] Run `make package APPLY=1`
- [ ] Verify `dist/` contains valid artifacts

## 4. Checksum/Manifest Gate
- [ ] Run `make package-check`
- [ ] Verify checksums and manifest are correct

## 5. Tag Gate
- [ ] Tag the release: `git tag -s v0.1.3 -m "ZAI Coder v0.1.3 patch release"`
- [ ] Verify tag signature: `git tag -v v0.1.3`

## 6. GitHub Release Gate
- [ ] Create release draft on GitHub
- [ ] Link changelog and evidence docs

## 7. Asset Upload Gate
- [ ] Upload `dist/` artifacts
- [ ] Upload `RELEASE_MANIFEST.json`

## 8. Post-release Verification Gate
- [ ] Run install/uninstall diagnostics against new tag/release

## 9. Branch Cleanup Gate
- [ ] Merge release branch to main
- [ ] Delete release branch

## 10. Stop Conditions
- Any failed validation test suite.
- Unexpected file mutation in `dist/` or `data/` directories.
- Mismatched checksums during package check.
- GPG tag verification failure.
