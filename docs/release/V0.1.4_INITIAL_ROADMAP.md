# ZAI Coder v0.1.4 Initial Roadmap

## Recommended PR Sequence

1. `chore/v0.1.4-cli-diagnostics-plan`
   - inventory CLI commands
   - define diagnostics behavior
   - add tests for help/version/error output

2. `test/v0.1.4-policy-safe-runner`
   - extend safe-runner and policy regression coverage
   - verify blocked command UX

3. `test/v0.1.4-installer-reliability`
   - add installer/update/uninstall tests
   - verify post-install diagnostics

4. `chore/v0.1.4-release-readiness`
   - add or improve release readiness status script
   - add release precondition tests

5. `docs/v0.1.4-operator-docs`
   - improve README / docs index / troubleshooting examples

## Principles

- Small PRs.
- One concern per PR.
- No direct main commits.
- No release mutation outside publish phase.
- Keep generated state untracked.
- Keep full validation green.
