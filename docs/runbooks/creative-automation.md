# Runbook — Creative Automation

## Purpose

Use the creative automation layer to plan, validate, and export creative projects.

## Flow

1. Create project metadata.
2. Register assets.
3. Draft content.
4. Move to review.
5. Approve.
6. Generate export plan.
7. Review generated files/commands.
8. Apply manually or through future APPLY=1 workflow.

## Safety

- Dry-run-first.
- Local-only by default.
- No external posting/rendering by default.
- Original content only.
- Block unsafe paths.
